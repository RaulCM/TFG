RESUMEN

PEP8-Analyzer es una aplicación web dirigida a analizar proyectos desarrollados en Python para comprobar su nivel de cumplimiento de la guía de estilo PEP8 y corregir algunos de los errores detectados.
Para realizar el análisis se hace uso de la herramienta Pylint, que analiza individualmente los ficheros e indica qué normas de la guía de estilo PEP8 se están incumpliendo.
Una vez obtenido el resultado del análisis realizado por Pylint, la aplicación web corrige los errores indicados en el listado de errores corregibles y realiza una Pull Request/Merge Request sobre el repositorio introducido con las modificaciones que se han realizado y un resumen de los errores corregidos situado en la descripción.

GUÍA DE USO

PEP8-Analyzer está divido en las siguientes páginas:

  • Inicio
  • Lista de repositorios
  • Lista de errores
  • Guía
  • Contacto

Inicio

En la pantalla de inicio se muestra un único formulario en el que se puede introducir el enlace al proyecto que se quiere analizar.
Para analizar un proyecto éste debe estar alojado en un repositorio de GitHub o del GitLab de la ETSIT de la URJC.
Una vez introducido el enlace al repositorio que contiene el proyecto, se muestra un breve resumen de los datos del repositorio, indicando su nombre, propietario, descripción y enlace, y un botón para ejecutar el análisis con Pylint.
Tras ejecutar Pylint, se muestra el resultado del análisis, pudiendo escoger visualizar todos los errores detectados por Pylint o filtrarlos para ver sólo los que son corregibles por la aplicación web, y un botón con el que corregir dichos errores.
Después de corregir los errores, se realiza la Pull Request/Merge Request y se muestran los datos del repositorio junto al enlace a la Pull Request/Merge Request que se ha realizado.
Dicha Pull Request/Merge Request deberá ser aceptada o cancelada por el usuario propietario del repositorio original.
