SUMMARY

PEP8-Analyzer is a web application aimed at analyzing Python projects to check their compliance level with the PEP8 style guide and correct some of the errors detected.
The analysis is performed by Pylint tool, which analyzes the files individually and indicates which rules of the PEP8 style guide are not being complied.
Once the result of the analysis performed by Pylint is obtained, the web application corrects the errors indicated in the fixable errors list and performs a Pull Request/Merge Request on the entered repository with the modifications that have been made and a summary of the corrected errors located in the description.

USE GUIDE

PEP8-Analyzer is divided into the following pages:

  • Home
  • Pull-Requests
  • Errors
  • Guide
  • Contact

Home

On the Home page there is a single form where the link to the project to be analyzed can be entered.
To analyze a project it must be hosted in a GitHub or URJC ETSIT GitLab repository.
Once the link to the repository containing the project is entered, a brief summary of the repository data is displayed, indicating its name, owner, description and URL, and a button to run the Pylint analysis.
After running Pylint, the result of the analysis is displayed with the possibility of viewing all the errors detected by Pylint or filter them to see only those that can be corrected by the web application, and a button to correct those errors.
After correcting the errors, the Pull Request/Merge Request is commited and the repository data is displayed together with the link to the Pull Request/Merge Request that has been commited.
Such Pull Request/Merge Request must be accepted or cancelled by the original repository owner.

Pull-Requests

Pull-Requests shows a list of Pull Requests/Merge Requests commited by the application and pending approval by the repository owner.

Errors

Two graphs are shown in the Error List.
The first graph shows the number of times that each fixed error that have occurred in the analyzed repositories .
The second graph shows the number of analyzed repositories and separates them according to the status of the Pull Request/Merge Request.

Guide

This page contains a summary of the application, a brief user guide defining each of the pages and a table with the errors detected by Pylint that can be fixed by the application.
